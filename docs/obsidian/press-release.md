# Press Release: 2ndbrain: Make Your Obsidian Vault AI-Native in Place

## Press Release

**A native command-line interface and Model Context Protocol server that adds a local semantic search, RAG chat, and link intelligence layer directly to your existing Obsidian vault.**

---

AUSTIN, TX, June 5, 2026: Today, Apres AI announced 2ndbrain version 0.5.0, a native command-line interface (CLI) and Model Context Protocol (MCP) server designed to make existing Obsidian vaults instantly AI-native. Rather than requiring users to import their documents into a separate system, 2ndbrain operates directly on any standard Obsidian directory in place. The tool adds local semantic search, retrieval-augmented generation (RAG) chat, and link graph intelligence to the knowledge base developers already have, without altering their files or introducing vendor lock-in.

### The Problem

Developers maintain critical technical knowledge in markdown: architecture decision records, runbooks, API documentation, project notes, and changelogs. But the tools that edit this knowledge and the AI agents that consume it live in separate worlds.

When an AI coding assistant needs to understand your project, it typically reads files with grep. It searches by keyword, not meaning. It cannot ask "what decisions did we make about authentication?" and get a ranked answer. It can only match the exact string "authentication" across thousands of files, consuming tokens on irrelevant matches. In a benchmark on a 155,000-line codebase, vanilla grep-based search spawned 5 subagents and consumed massive token budgets. A local semantic search layer reduced input tokens by 97 percent: same results, fraction of the cost.

Existing solutions for bridging this gap require importing markdown files into proprietary formats, injecting unwanted metadata like stable UUIDs into frontmatter, or running complex, brittle community-built plugin integrations. Developers love Obsidian because it is local-first, uses plain markdown files, and guarantees that their data is theirs. However, Obsidian lacks a native CLI, a built-in MCP server, and a structured output format for AI tools.

### The Solution

2ndbrain version 0.5.0 introduces a clean break from the standalone editor model. It repositions 2ndbrain as a lightweight, non-mutating AI companion for your Obsidian vault. It detects vaults using standard `.obsidian/` directory markers and writes all index and embedding data to a gitignored `.2ndbrain/` sidecar folder. It guarantees non-mutating file access: 2ndbrain never injects identifiers, never rewrites frontmatter properties, and leaves the user's raw files completely untouched.

The system features three interfaces:

**A powerful Go-based CLI (`2nb`)** that handles indexing, semantic search, and RAG operations. Every command supports structured JSON output. For example, `2nb search "JWT refresh strategy" --json` returns ranked results with relevance scores, matching text chunks, and metadata.

**A built-in MCP server** that exposes your vault as searchable resources to any MCP-compatible AI assistant. Claude Code, Cursor, and Windsurf connect directly. When your AI assistant needs context, it queries the local database semantically, retrieving exact matches without manual searching.

**A native macOS config GUI** that replaces the legacy editor. The macOS app now acts as a control center to configure AI providers and models (AWS Bedrock by default, with optional local Ollama), monitor vault indexing status, manage the MCP connection, and view diagnostic logs. Editing remains in Obsidian.

"We realized that developers do not want a new editor; they want their existing Obsidian vault to speak to their AI tools," said Chad Neal, founder of Apres AI. "With 2ndbrain 0.5.0, your vault becomes your AI assistant's memory. It indexes your documents locally, respects your formatting, and exposes a high-fidelity semantic interface to your workspace. Your agent finally has the context to write correct code on the first try."

### How It Works

**Zero-Configuration Indexing:** Run `2nb index` in any Obsidian vault. The CLI detects the `.obsidian/` marker and builds a hybrid index (BM25 keyword search combined with vector embeddings generated by your chosen provider: AWS Bedrock by default, or local Ollama if you opt in).

**Non-Mutating Security:** Write your notes using your preferred Obsidian editor. 2ndbrain reads your files but never inserts fields, preserves all YAML comments, and respects key ordering.

**Instant Agent Context:** Add `2nb mcp-server` to your AI client configuration. The assistant gains direct access to your notes, enabling it to answer questions, locate related concepts, and write code based on your local style and documentation.

"Before 2ndbrain, I had to copy-paste code guidelines and architecture decision records into every prompt," said a beta tester. "Now, Claude Code searches my Obsidian vault automatically. It knows which authentication approach we decided on in our ADRs and respects those boundaries without my intervention."

### Key Features

* **Obsidian-Native substrate:** Works directly on standard Obsidian vaults. Detects `.obsidian/` and uses `.2ndbrain/` as a gitignored sidecar.
* **Non-mutating guarantee:** Never injects UUIDs or alters file frontmatter. Preserves all layout, key ordering, and comments.
* **Path-based identity:** Files are identified by their vault path. Renames are tracked via a local surrogate key database to avoid re-embedding.
* **Local index, your choice of AI:** The search index is a local SQLite database on your machine. Embeddings and RAG answers are produced by the AI provider you configure: AWS Bedrock by default (using your own AWS account), or fully on-device with opt-in local Ollama models.
* **Structured CLI:** Commands output JSON, YAML, or CSV. Scriptable and friendly to automated workflows.
* **Model Context Protocol:** Out-of-the-box MCP server for seamless integration with modern AI coding tools.

### Availability

2ndbrain version 0.5.0 is available as an open-source project at github.com/apresai/2ndbrain. Both the `2nb` CLI and the macOS configuration app are distributed via Homebrew for macOS (Apple Silicon and Intel): `brew install apresai/tap/2nb` for the CLI and the `secondbrain` cask for the app.

## Frequently Asked Questions

### External FAQ (Customer Questions)

#### Does 2ndbrain require a subscription or internet connection?
2ndbrain is free and open-source, with no subscription. The index and search run locally and the search index is stored in a local SQLite database. By default, AI features (embeddings and RAG answers) use AWS Bedrock through your own AWS account, which requires AWS credentials and an internet connection. To run fully offline, enable the opt-in local Ollama provider in `2nb ai setup` or the macOS AI Hub.

#### Will 2ndbrain modify my markdown notes?
No. In version 0.5.0, 2ndbrain guarantees non-mutating access. It does not write to, inject fields into, or modify your markdown files in any way during indexing or search.

#### How does 2ndbrain identify notes without UUIDs?
2ndbrain uses the relative file path within the vault as the canonical identity. To handle renames without losing embeddings, 2ndbrain maps paths to an internal surrogate key in its local sidecar database.

### Internal FAQ (Engineering / Business Questions)

#### Why pivot away from the native Swift editor?
Developing a fully featured markdown editor to compete with Obsidian requires significant engineering effort. By pivoting to a CLI/MCP companion, we focus entirely on the AI integration layer where 2ndbrain provides the highest value, while allowing developers to use their preferred editor.

#### How do we handle concurrent access to the SQLite index?
The local SQLite database (`.2ndbrain/index.db`) is configured with Write-Ahead Logging (WAL) enabled. This allows concurrent read access by the CLI, MCP server, and macOS GUI while writes are in progress.
